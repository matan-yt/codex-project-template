# Project Spec Template

Use this to define scope before implementation.

## Goal

- What are we trying to achieve?
- What problem does this solve?
- What does success look like in 1-2 measurable outcomes?

## Users

- Primary user type(s).
- Key user jobs-to-be-done.
- Any constraints (technical skill, device, location, compliance).

## MVP features

- List only must-have features for first release.
- Keep each item testable and outcome-focused.
- Mark dependencies or sequencing if relevant.

## Non-goals

- Explicitly list what is out of scope for MVP.
- Call out tempting features deferred to later phases.

## Screens / UI

- List core screens/views and their purpose.
- Note main actions per screen.
- Include rough wireframes or links when available.

## Data model

- Core entities and key fields.
- Relationships between entities.
- Basic validation/business rules.

## Definition of done

- Functional acceptance criteria met.
- Tests added and passing.
- Docs/env/setup updated.
- Deployed to target environment and verified.
