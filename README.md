# Repository Template

A clean starting point for new projects.

## Use this template

1. Copy this folder or create a new repo from it.
2. Rename the project in this `README.md`.
3. Fill `.env.example` with required variables (placeholders only).
4. Add your first decision to `docs/decision-log.md`.
5. Add startup/ops steps to `docs/runbook.md`.
6. Start building app code in a new `src/` folder.

## First-day checklist

- Update this README with project goals.
- Set up language/tooling (package manager, formatter, linter, tests).
- Add CI checks.
- Create initial release note entry in `CHANGELOG.md`.

## Template structure

- `AGENTS.md` - working rules for contributors and AI agents
- `CHANGELOG.md` - human-readable change history
- `.env.example` - environment variable placeholders
- `docs/decision-log.md` - short architecture decisions
- `docs/runbook.md` - operational notes and run steps
- `scripts/` - automation scripts
